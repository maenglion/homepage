Roadmap & Milestones

우리가 걸어온 길

CASP ENGINE은 “어떻게 말했는가”를 데이터로 만들기 위해, 대화 기반 리포트 → 음성 기반 비언어 지표 분석 → 프로토콜화의 순서로 발전해왔습니다.

    Engine Core Indexing Integration-ready SDK / API

CASP Protocol v1 — SQ 지수와의 연결

CASP Protocol v1은 단순 분석 결과가 아니라, 반복 측정 가능한 지표 체계를 만드는 데 목적이 있습니다. 이 과정에서 SQ 지수를 “커뮤니케이션/시스템화 특성”의 기준축으로 활용하여, 음성 기반 지표와의 연관을 검증할 수 있도록 설계했습니다.

1) 입력(데이터)

세션 기반 음성 수집(상황/과제 단위) + 기본 설문(예: SQ 지수)을 결합하여 상태/특성의 기준점을 확보합니다.

2) 품질 게이트

SNR, 녹음 길이, 말-침묵 비율 등 품질 기준으로 분석 가능 여부를 먼저 판정합니다.

3) 지표 산출 & 연결

말속도/멈춤/리듬/변동성 지표를 산출하고, SQ 지수와의 관계를 탐색 가능한 형태로 인덱싱합니다.

왜 SQ 지수인가

SQ 지수는 개인의 시스템화 경향(구조화/규칙성 선호)을 반영하는 축으로, 발화 패턴의 구조성·일관성·전환 양상과 함께 해석될 때 커뮤니케이션 특성의 설명력을 높일 수 있습니다.
